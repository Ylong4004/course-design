# 1 题目简介与设计目的

撰写人：架构设计负责人 光雨晨

## 1.1 题目简介

本系统面向区域城市交通网络开展建模与综合分析。将参与路网的每一座城市抽象为图的**顶点**，城市之间连通的道路抽象为**带权边**，道路长度、通行耗时等作为边的**权值**，支持单向/双向通行，灵活构建有向带权图或无向带权图。

系统同时使用**邻接矩阵**和**邻接表**两种图存储结构，集成**深度优先遍历（DFS）**、**广度优先遍历（BFS）**、**Dijkstra 最短路径**、**Floyd 多源最短路径**、**Prim 最小生成树**、**Kruskal 最小生成树**、**拓扑排序**七大经典图算法，并手写实现了**循环队列**、**顺序栈**、**最小堆优先队列**、**并查集**四种基础数据结构。系统支持路网编辑、最短路径规划、最小生成树构建、拥堵仿真模拟、存储结构性能对比、数据持久化等全维度分析功能。

## 1.2 设计目的

1. **知识巩固**：综合运用线性表、栈、队列、树、图、排序、查找等数据结构核心知识，结合城市交通路网实际业务场景进行实践应用。
2. **算法能力训练**：手写实现图论经典算法，掌握 DFS、BFS、Dijkstra、Floyd、Prim、Kruskal、拓扑排序的算法原理与代码实现，具备算法复杂度分析能力。
3. **工程实践能力**：采用模块化设计思想，完成系统设计→编码→测试→文档全流程，体验多人协作开发模式。
4. **数据结构对比认知**：通过邻接矩阵与邻接表的双实现并行对比，直观理解不同存储结构在内存占用、遍历速度、查找效率方面的差异与适用场景。

# 2 问题描述与需求分析

撰写人：架构设计负责人 光雨晨

## 2.1 问题详细描述

本系统面向区域城市交通网络建模，需要对城市和道路进行抽象建模并开展多维度的网络分析。具体而言：

- **路网建模**：城市顶点支持新增和删除，道路边支持新增、删除和权值修改。所有修改需在邻接矩阵和邻接表两套存储结构上**同步执行**，保证数据一致性。
- **图遍历**：使用 DFS 和 BFS 两种方式遍历整个交通网络，输出城市访问序列。需正确处理非连通图场景。
- **最短路径规划**：Dijkstra 算法计算单源最短路径（指定起点到其余所有城市的最优路线与总路程）；Floyd 算法计算多源最短路径（输出任意两城间的最短距离对照表）。
- **最小生成树**：Prim 和 Kruskal 两种算法分别构建路网最小生成树，计算连通所有城市的最低道路建设总造价。
- **拓扑排序与环路检测**：对有向路网执行拓扑排序，输出合法通行序列，检测环路以判断是否会引发交通死锁。
- **拥堵仿真**：修改指定道路权值以模拟拥堵/限速，对比拥堵前后最优路线变化。
- **性能对比**：统计两种存储结构在内存占用、遍历耗时、查找效率等方面的差异。
- **数据持久化**：路网数据以 TXT 文件保存，程序启动自动加载，支持多路网文件切换。

## 5.2 核心算法设计与伪代码

### 5.2.1 Dijkstra 最短路径

```
算法 Dijkstra(G, start):
  输入: 图 G, 起点 start
  输出: dist[v] = start 到 v 的最短距离, prev[v] = v 的前驱

  for 每个顶点 v:
      dist[v] ← ∞,  prev[v] ← -1,  determined[v] ← false
  dist[start] ← 0

  for count = 1 to V:
      u ← 未确定顶点中 dist 最小的顶点
      if u == -1: break           // 剩余顶点不可达
      determined[u] ← true
      for u 的每个邻接顶点 v:
          if dist[u] + weight(u,v) < dist[v]:
              dist[v] ← dist[u] + weight(u,v)   // 松弛
              prev[v] ← u

  return dist, prev
```

### 5.2.2 Floyd 多源最短路径

```
算法 Floyd(G):
  输入: 图 G（V 个顶点）
  输出: dist[i][j] = i→j 最短距离, next[i][j] = i→j 路径上的下一个顶点

  for i = 0 to V-1:
      for j = 0 to V-1:
          if 边(i,j) 存在: dist[i][j] ← weight(i,j), next[i][j] ← j
          else: dist[i][j] ← ∞, next[i][j] ← -1
      dist[i][i] ← 0

  for k = 0 to V-1:               // 以 k 为中转点
      for i = 0 to V-1:
          for j = 0 to V-1:
              if dist[i][k] + dist[k][j] < dist[i][j]:
                  dist[i][j] ← dist[i][k] + dist[k][j]
                  next[i][j] ← next[i][k]
```

### 5.2.3 Prim 最小生成树

```
算法 Prim(G):
  for 每个顶点 v: key[v] ← ∞, parent[v] ← -1, inMST[v] ← false
  key[0] ← 0
  for count = 1 to V:
      u ← 未加入 MST 中 key 最小的顶点
      inMST[u] ← true
      for u 的每个邻接点 v:
          if !inMST[v] and weight(u,v) < key[v]:
              key[v] ← weight(u,v), parent[v] ← u
  根据 parent 生成边集
```

### 5.2.4 Kruskal 最小生成树

```
算法 Kruskal(G):
  收集图中所有边 → 按权值升序排序
  初始化并查集：每个顶点自成一个集合
  for 每条边 e = (u, v, w)（按序）:
      if find(u) ≠ find(v):
          union(u, v)
          将 e 加入 MST
          if |MST| == V-1: break
  if |MST| != V-1: return 图不连通
```

### 5.2.5 拓扑排序（Kahn 算法）

```
算法 Kahn(G)（G 须为有向图）:
  计算每个顶点的入度 indegree[]
  for 每个顶点 v:
      if indegree[v] == 0: queue.push(v)
  while queue 非空:
      v ← queue.pop()
      将 v 加入拓扑序列
      for v 的每个邻接点 w:
          indegree[w]--
          if indegree[w] == 0: queue.push(w)
  if 序列长度 < V: return 存在环路
  else: return 拓扑序列
```

## 5.3 关键业务逻辑实现思路

### 双存储同步

所有路网编辑操作（增删城市/增删改道路）均在 `RoadNetwork` 类中通过 `matrix_graph` 和 `list_graph` 两个 `GraphBase*` 指针同步执行。例如 `add_city` 同时调用 `matrix_graph->add_vertex(city)` 和 `list_graph->add_vertex(city)`，任一失败则回滚。

### 算法与存储解耦

所有算法函数接受 `const GraphBase* graph` 参数，通过 `get_vertex_count()`、`get_neighbors()` 等虚函数操作图数据，不感知底层是邻接矩阵还是邻接表。同一份算法代码可复用于两种存储结构。

### 拥堵模拟对比

`CongestionSimulator` 在修改权值前备份原始值。`run_comparison` 分别在拥堵前后的路网上运行 Dijkstra，缓存两组距离数组，输出对比报告（哪些城市的路径发生变化、距离增加多少）。

## 6.4 异常处理与容错设计

| 异常类别 | 具体场景 | 处理方案 |
|----------|----------|----------|
| 输入异常 | 非法整数、越界范围、空字符串 | `read_int_safe` 带重试循环，`read_str_safe` 长度检查 |
| 业务异常 | 重复编号、不存在的城市、自环边、负权值、道路重复 | 返回具体错误码，菜单显示对应中文提示 |
| 文件异常 | 文件不存在、格式错误、权限不足 | 返回 `ERR_FILE_OPEN_FAIL`/`ERR_FILE_FORMAT`，不崩溃 |
| 空数据 | 空图执行算法 | Dijkstra 返回 `ERR_GRAPH_EMPTY`，遍历输出空序列 |
| 有向/无向冲突 | 拓扑排序用于无向图、MST 用于有向图 | 预先判断图类型，给出"仅适用于有向/无向图"提示 |
| 内存不足 | new 返回 nullptr | `safe_new` 宏检测失败后打印错误信息并 `exit` |

## 6.5 文件读写与数据持久化

**文件格式（TXT）**：

```
图类型 城市数 道路数
城市ID 城市名称
...
起点ID 终点ID 权值
...
```

- **图类型**：0 = 无向图，1 = 有向图
- 自动加载：程序启动时优先加载 `data/default.txt`，其次加载 `data/` 下第一个 `.txt`
- **多路网切换**：列出 `data/*.txt` 供用户选择，选择后替换当前路网
- **自动保存**：退出系统时自动将当前路网保存到当前加载的文件
- **异常处理**：文件打开失败、格式错误均返回对应错误码，不影响程序运行

# 8 算法复杂度分析

撰写人：算法 & 核心功能负责人 柳浪一

| 算法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| DFS（递归） | O(V+E) | O(V) | visited 数组 + 递归栈 |
| BFS（队列） | O(V+E) | O(V) | visited 数组 + 队列 |
| Dijkstra | O(V²) | O(V) | 数组实现，每次选最小遍历 V 个顶点 |
| Dijkstra（优化） | O((V+E)·logV) | O(V) | 最小堆优先队列 |
| Floyd | O(V³) | O(V²) | 三重循环 + 距离/路径矩阵 |
| Prim | O(V²) | O(V) | 数组实现 |
| Prim（优化） | O((V+E)·logV) | O(V) | 最小堆优先队列 |
| Kruskal | O(E·logE) | O(V) | 排序主导，并查集 O(α(V)) |
| Kahn 拓扑排序 | O(V+E) | O(V) | 入度表 + 队列 |
| 邻接矩阵边查询 | O(1) | — | 二维数组直接索引 |
| 邻接表边查询 | O(degree) | — | 遍历链表 |
| 并查集 Find/Union | O(α(V)) ≈ O(1) | O(V) | 路径压缩 + 按秩合并 |

# 9 系统评价与总结

撰写人：交互 & 测试 & 汇总负责人 （待填写姓名）

## 9.1 程序优点

1. **架构清晰**：采用分层设计，算法层通过抽象接口与存储层解耦，同一份算法代码可运行于邻接矩阵和邻接表两种结构。
2. **功能完备**：实现了选题要求的全部 9 项基础功能，并拓展了路网详情查看、自动保存、调试模式、自动化测试等功能。
3. **基础结构手写**：队列、栈、并查集、最小堆优先队列全部从零实现，不使用 STL 容器替代。
4. **代码规范**：统一命名规则（snake_case）、Doxygen 注释全覆盖、安全内存宏统一管理。
5. **测试充分**：23 个自动化测试用例覆盖合法/非法/边界/交叉验证五类场景。
6. **用户体验**：具体错误提示替代笼统的失败消息；文件路径简化为只填文件名。
7. **数据持久化**：文件化管理路网，支持多路网切换和退出自动保存。

## 9.2 程序存在不足

1. **图类型固定**：当前无向图/有向图由初始化时确定，未提供运行时切换功能。
2. **可视化缺失**：未实现路网的图形化展示，不利于直观理解图结构。
3. **Dijkstra 仅为基本版**：未启用 PriorityQueue 优化版，时间复杂度为 O(V²)。

## 9.3 优化设想与拓展方案

1. 增加图类型运行时切换（无向 ↔ 有向）
2. Dijkstra/Prim 加入 PriorityQueue 优化版，降低稀疏图时间开销
3. 增加命令行交互模式（如输入 `dijkstra 1` 直接执行）
4. 使用 ASCII 字符绘制简单路网拓扑图
5. 增加按城市名称查询的功能
